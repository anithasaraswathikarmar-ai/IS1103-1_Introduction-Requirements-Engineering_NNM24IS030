# CampusKart – SDLC Comparative Study

This repository contains the Software Engineering Assignment for Unit I:
Software Process Models and Requirements Engineering.

## Case Study:
CampusKart – An Online E-Commerce Platform for College Students

## Contents:

- SDLC comparative report (Waterfall, Incremental, Spiral)
- Initial and Revised SRS documents
- Diagrams of process models

## Branching Strategy:

We followed a simple Git branching strategy:

main branch:
Contains final approved report and revised SRS.

feature/requirements-v1:
Contains initial SRS document.

feature/requirements-revision:
Contains updated requirements after stakeholder validation.

## Merging Strategy:

- Initial SRS was committed in feature branch.
- After review, revised SRS was updated in a separate branch.
- Pull Request was created and merged into main after approval.

## Tools Used:

- MS Word (SRS documentation)
- Draw.io (Diagrams)
- GitHub for version control
